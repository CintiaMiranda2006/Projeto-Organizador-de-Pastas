# PRD — Sistema de Login

## Visão geral

Este documento descreve os requisitos do sistema de autenticação de usuários.

## Funcionalidades

- Login com email e senha.
- Recuperação de senha por email.
- Bloqueio após 5 tentativas incorretas.
- Token de sessão com expiração de 24h.

## Regras de negócio

- A senha deve ter no mínimo 8 caracteres.
- O email deve ser único no sistema.
- Tentativas de login devem ser registradas em log.

## Critérios de aceite

- Usuário consegue se autenticar com credenciais válidas.
- Usuário recebe mensagem de erro com credenciais inválidas.
- Sistema envia email de recuperação em até 2 minutos.
